Сжатие изображений SVD и JPEG

Это учебный проект на языке C. В проекте реализовано сжатие изображений двумя способами: через SVD и через JPEG-подобный алгоритм.

Главная цель проекта: сравнить два метода по качеству изображения, размеру результата и времени работы.

Что делает программа

Программа умеет читать BMP-изображение, сжимать его методом SVD, сохранять результат SVD в файл .svd, восстанавливать BMP из .svd, сжимать изображение JPEG-подобным методом, сохранять результат в .jpg, считать PSNR и SSIM, а также сравнивать несколько параметров сжатия через режим compare.

Как работает SVD

В SVD-сжатии изображение рассматривается как матрица пикселей.

Матрица изображения раскладывается на три матрицы:

A = U * S * V^T

После этого сохраняются не все значения, а только первые k сингулярных значений.

Если k маленькое, файл получается меньше, но качество изображения хуже. Если k большое, качество лучше, но размер файла больше.

Пример запуска SVD:

image_compressor svd input.bmp out_svd.svd 20

Чтобы посмотреть результат, нужно восстановить BMP:

svd_to_bmp out_svd.svd out_svd.bmp

Как работает JPEG

JPEG-режим сделан по основным шагам JPEG-сжатия.

Сначала изображение делится на блоки 8 на 8 пикселей. Потом для каждого блока считается DCT. После этого коэффициенты квантуются. Затем изображение восстанавливается обратно и сохраняется в .jpg.

Для JPEG используется параметр quality.

Чем больше quality, тем лучше качество и больше размер файла. Чем меньше quality, тем сильнее сжатие и хуже качество.

Пример запуска JPEG:

image_compressor jpeg input.bmp out_jpeg.jpg 60

Сборка

Для сборки нужен GCC.

Windows

Можно собрать проект через build.bat:

build.bat

После сборки должны появиться файлы:

image_compressor.exe
svd_to_bmp.exe

Также можно запускать сборку из VS Code через Ctrl + Shift + B.

Linux или macOS

Для сборки нужно выполнить:

make

После сборки должны появиться файлы:

image_compressor
svd_to_bmp

Для очистки файлов сборки:

make clean

Запуск программы

Общий вид команд:

image_compressor svd input.bmp output.svd k
image_compressor jpeg input.bmp output.jpg quality
image_compressor compare input.bmp out_dir k_list quality_list

Утилита для восстановления SVD:

svd_to_bmp input.svd output.bmp

Примеры для Windows

SVD:

.\image_compressor.exe svd .\app.bmp .\out_svd.svd 20

Восстановление SVD в BMP:

.\svd_to_bmp.exe .\out_svd.svd .\out_svd.bmp

JPEG:

.\image_compressor.exe jpeg .\app.bmp .\out_jpeg.jpg 60

Сравнение:

.\image_compressor.exe compare .\app.bmp .\results 5,10,20,40 20,40,60,80

Примеры для Linux/macOS

SVD:

./image_compressor svd ./app.bmp ./out_svd.svd 20

Восстановление SVD в BMP:

./svd_to_bmp ./out_svd.svd ./out_svd.bmp

JPEG:

./image_compressor jpeg ./app.bmp ./out_jpeg.jpg 60

Сравнение:

./image_compressor compare ./app.bmp ./results 5,10,20,40 20,40,60,80

Параметры

Параметр k

k используется в SVD.

Примерные значения:

5       сильное сжатие, качество плохое
10-20   средний вариант
40+     качество лучше, но файл больше

Точный результат зависит от изображения.

Параметр quality

quality используется в JPEG.

Примерные значения:

20      сильное сжатие, могут быть заметные искажения
40-60   средний вариант
80+     хорошее качество, но файл больше

Метрики

После сжатия программа выводит статистику.

Пример:

PSNR : 32.1234 dB
SSIM : 0.912345
Ratio: 4.5678
Time : 0.123456 s

PSNR показывает отличие от исходного изображения. Чем больше значение, тем лучше.

SSIM показывает похожесть изображений. Чем ближе значение к 1, тем лучше.

Ratio показывает степень сжатия.

Time показывает время работы программы.

Режим compare

Режим compare нужен, чтобы не запускать программу много раз вручную.

Пример:

./image_compressor compare ./app.bmp ./results 5,10,20,40 20,40,60,80

После запуска в папке results появятся файлы с результатами и таблица summary.csv.

Пример содержимого папки:

results/
  svd_k5.svd
  svd_k10.svd
  svd_k20.svd
  svd_k40.svd
  jpeg_q20.jpg
  jpeg_q40.jpg
  jpeg_q60.jpg
  jpeg_q80.jpg
  summary.csv

В summary.csv записываются значения PSNR, SSIM, степени сжатия и времени работы.

Что можно сравнивать

При сравнении SVD и JPEG можно смотреть на качество изображения, размер полученного файла, время работы программы, значения PSNR и SSIM, а также визуальные отличия от исходного изображения.

Обычно SVD хорошо показывает математическую идею сжатия через матрицы, но JPEG лучше подходит для обычного сжатия изображений.

Возможные проблемы

Не найден gcc

Нужно проверить, установлен ли GCC:

gcc --version

Если команда не работает, надо установить MinGW/MSYS2 на Windows или GCC через пакетный менеджер на Linux/macOS.

Не найден make

На Windows можно использовать build.bat.

На Linux/macOS нужно установить make.

Не открывается файл .svd

.svd это не картинка, а внутренний файл программы. Чтобы посмотреть изображение, надо восстановить BMP:

svd_to_bmp out_svd.svd out_svd.bmp

Программа не читает изображение

Нужно проверить, что файл действительно BMP и путь указан правильно.

Что можно доделать

В будущем можно добавить поддержку PNG или JPG на входе, графики по результатам из summary.csv, больше тестов, CMake для сборки и более удобный вывод результатов.

Вывод

В этом проекте реализованы два способа сжатия изображений: SVD и JPEG-подобный алгоритм. Программа позволяет запускать оба метода, менять параметры сжатия и сравнивать результат по качеству, размеру файла и времени работы.
