# SVD--: сжатие изображений на C

Проект реализует и сравнивает два подхода к сжатию изображений на языке C:

- **SVD-сжатие** — сохранение изображения через усечённое сингулярное разложение матрицы пикселей;
- **JPEG-подобное сжатие** — обработка блоков изображения через DCT, квантование, восстановление и запись результата в JPEG-файл.

Программа работает с BMP-изображениями на входе, умеет сохранять результат SVD в собственный бинарный формат `.svd`, сохранять JPEG-результат в `.jpg`, а также считать метрики качества: **PSNR**, **SSIM**, коэффициент сжатия и время выполнения.

---

## Возможности

- чтение исходного изображения из BMP;
- сжатие изображения методом SVD с параметром `k`;
- сохранение SVD-результата в компактный бинарный файл `.svd`;
- восстановление BMP из `.svd` отдельной утилитой `svd_to_bmp`;
- JPEG-подобное сжатие с параметром качества `quality`;
- сохранение JPEG-результата в настоящий файл `.jpg`;
- расчёт метрик качества:
  - `PSNR` — пиковое отношение сигнал/шум;
  - `SSIM` — индекс структурного сходства;
  - `Ratio` — отношение размера исходного файла к размеру результата;
  - `Time` — время выполнения сжатия;
- пакетный режим `compare`, который прогоняет несколько значений `k` и `quality` и сохраняет CSV-отчёт.

## Как это работает

### SVD-сжатие

Изображение рассматривается как матрица пикселей `A`. Для неё выполняется сингулярное разложение:

```text
A = U * S * V^T
```

где:

- `U` и `V` — ортогональные матрицы;
- `S` — диагональная матрица сингулярных значений;
- `k` — количество крупнейших сингулярных значений, которые сохраняются.

Чем больше `k`, тем выше качество восстановленного изображения, но тем больше размер `.svd`-файла. Чем меньше `k`, тем сильнее сжатие, но заметнее потери качества.

Пример:

```bash
image_compressor svd input.bmp out_svd.svd 20
```

После этого результат можно восстановить в BMP:

```bash
svd_to_bmp out_svd.svd out_svd.bmp
```

### JPEG-подобное сжатие

JPEG-режим использует классический принцип JPEG-сжатия:

1. изображение делится на блоки 8x8 пикселей;
2. к каждому блоку применяется прямое дискретное косинусное преобразование, DCT;
3. DCT-коэффициенты квантуются;
4. выполняется обратное преобразование для оценки качества восстановленного изображения;
5. результат сохраняется в `.jpg`.

Параметр `quality` задаёт уровень качества JPEG-сжатия. Обычно он находится в диапазоне от `1` до `100`:

- низкое значение даёт меньший файл, но больше искажений;
- высокое значение даёт лучшее качество, но больший размер файла.

Пример:

```bash
image_compressor jpeg input.bmp out_jpeg.jpg 60
```

## Параметры

### `k` для SVD

`k` — количество сингулярных значений, которые сохраняются при сжатии.

Примерные ориентиры:

| Значение `k` | Что ожидать |
|---:|---|
| `5` | сильное сжатие, заметная потеря деталей |
| `10-20` | среднее качество, хороший вариант для экспериментов |
| `40+` | выше качество, но больше размер `.svd`-файла |

Точное качество зависит от размера и содержания изображения.

### `quality` для JPEG

`quality` — уровень качества JPEG-сжатия.

Примерные ориентиры:

| Значение `quality` | Что ожидать |
|---:|---|
| `20` | маленький файл, заметные артефакты |
| `40-60` | баланс размера и качества |
| `80+` | хорошее качество, но больший файл |

## Результаты и метрики

После запуска программа выводит статистику в консоль:

```text
PSNR : 32.1234 dB
SSIM : 0.912345
Ratio: 4.5678
Time : 0.123456 s
```

Расшифровка:

| Метрика | Значение |
|---|---|
| `PSNR` | Чем выше, тем меньше отличие восстановленного изображения от исходного. Измеряется в dB. |
| `SSIM` | Чем ближе к `1`, тем более похожи изображения по структуре. |
| `Ratio` | Во сколько раз исходный файл больше результата. Например, `4.0` означает сжатие примерно в 4 раза. |
| `Time` | Время выполнения операции в секундах. |

## Режим сравнения `compare`

Режим `compare` удобен для экспериментов. Он запускает SVD для нескольких значений `k`, JPEG — для нескольких значений `quality`, а затем сохраняет результаты в отдельную папку.

Пример:

```bash
./image_compressor compare ./app.bmp ./results 5,10,20,40 20,40,60,80
```

После выполнения будут созданы файлы вида:

```text
results/
├── svd_k5.svd
├── svd_k10.svd
├── svd_k20.svd
├── svd_k40.svd
├── jpeg_q20.jpg
├── jpeg_q40.jpg
├── jpeg_q60.jpg
├── jpeg_q80.jpg
└── summary.csv
```

Файл `summary.csv` содержит таблицу:

```csv
method,parameter,psnr,ssim,ratio,time_seconds
svd,5,...
svd,10,...
jpeg,20,...
jpeg,40,...
```

Эту таблицу можно открыть в Excel, LibreOffice Calc, Google Sheets или обработать скриптом для построения графиков.

## Как интерпретировать сравнение

При сравнении SVD и JPEG стоит смотреть сразу на несколько показателей:

1. **Качество** — `PSNR` и `SSIM`.
2. **Степень сжатия** — `Ratio`.
3. **Скорость** — `Time`.
4. **Визуальный результат** — наличие размытия, потери деталей, блочных JPEG-артефактов.

Обычно:

- SVD хорошо показывает идею матричного приближения и позволяет наглядно управлять качеством через `k`;
- JPEG лучше подходит для практического сжатия фотографий, потому что использует блочное DCT-преобразование и квантование;
- при малых `k` SVD может сильно размывать изображение;
- при низком `quality` JPEG может давать блочные артефакты.

## Форматы файлов

### Входной файл

На вход подаётся BMP-изображение:

```text
input.bmp
```

### Выход SVD

SVD-режим сохраняет бинарный файл:

```text
out_svd.svd
```

Этот файл не является изображением, поэтому его нельзя открыть обычным просмотрщиком. Чтобы посмотреть результат, нужно восстановить BMP:

```bash
svd_to_bmp out_svd.svd out_svd.bmp
```

### Выход JPEG

JPEG-режим сохраняет файл:

```text
out_jpeg.jpg
```

Его можно открыть обычным просмотрщиком изображений.

## Частые проблемы

### `gcc` не найден

Проверьте, установлен ли GCC и добавлен ли он в `PATH`:

```bash
gcc --version
```

На Windows проще всего установить MSYS2/MinGW.

### `make` не найден

На Windows используйте `build.bat`. На Linux/macOS установите `make` через пакетный менеджер.

### Программа не читает изображение

Проверьте, что файл действительно имеет формат BMP и путь указан правильно.

Пример корректного запуска:

```bash
./image_compressor jpeg ./app.bmp ./out_jpeg.jpg 60
```

### `.svd` не открывается как картинка

Это нормально. `.svd` — внутренний бинарный формат проекта. Для просмотра восстановите BMP:

```bash
./svd_to_bmp ./out_svd.svd ./out_svd.bmp
```

### Выходное расширение отличается от указанного

В режиме `svd` программа сохраняет результат как `.svd`, а в режиме `jpeg` — как `.jpg`. Если передать другое расширение, программа может автоматически заменить его на подходящее.
